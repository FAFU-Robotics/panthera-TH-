# fafu_robot_sdk 技术文档

> 版本: `fafu_robot_python 0.1.0` / `panthera_motor` pybind11 绑定
> 适用硬件: Fafu 机械臂 (基于 Hightorque / Panthera-HT 调试板, 7 电机 = 6 关节 + 1 夹爪)
> 本文是面向开发者的架构与实现说明; 上手用法见各级 `README.md`。

---

## 1. 概述与定位

`fafu_robot_sdk` 是 Fafu 机械臂的完整软件栈, 把"USB 调试板串口协议"封装成
Piper / Realman 风格的高层机器人 API。它分两侧, 共用同一份底层协议代码:

| 侧 | 语言 | 产物 | 角色 |
|---|---|---|---|
| `fafu_robot_cpp/` | C++ | `panthera_motor.pyd` + `serial_cmake.dll` + `fafu_robot_sdk.lib` + 例程 exe | 协议层 + pybind11 绑定 + 原生 C++ SDK |
| `fafu_robot_python/` | Python | `FafuRobotController` 等高层类 | 用户日常使用入口 |

设计目标:

- **接口对齐 Piper**: `move_j` / `open_gripper` / `grasp` 等同名同语义, 便于迁移。
- **完全自包含**: 所有 C++ 源码 (协议层 + 三方串口库) 已 vendor 进 `fafu_robot_cpp/`, 不依赖外部 `Panthera-HT_SDK/` 即可构建。
- **双语对等**: Python 和原生 C++ 两套高层 API 一一对应, 调用同一份 `HightorqueSerial`, 行为 100% 一致。

## 2. 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│ 用户代码 (脚本 / 上层规划 / teleop / 视觉伺服)               │
└───────────────┬──────────────────────────┬──────────────────┘
                │ Python                    │ C++
                ▼                           ▼
   FafuRobotController (py)      fafu_robot::FafuRobotController (C++)
   fafu_robot_python/            fafu_robot_cpp/sdk/
   - 单位换算 rad↔turns          - 同名 API (MoveOpts/GripperOpts/...)
   - move_j / servo_j            - servoJ + 看门狗
   - 夹爪 / 力控抓取 grasp       - 关节空间 + 夹爪 (不含笛卡尔)
   - 软限位 / 急停
   - 动力学 (Pinocchio):
     FK/IK, 重力/摩擦补偿
                │ import panthera_motor      │ #include "fafu/..."
                ▼                           ▼
        panthera_motor.pyd           fafu_robot_sdk.lib
        (pybind11 绑定)               (C++ 静态库)
                └─────────────┬─────────────┘
                              ▼
        hightorque::HightorqueSerial  ← 调试板协议层 (vendored, 双侧共用)
        - 打开串口 / 异步 RX 线程
        - 后台状态轮询
        - set_pos_vel_acc / set_many_pos_vel_tqe / set_torque / ...
        - 软限位 clamp / 单位换算 to_turns/from_turns
                              ▼
              serial_cmake (.dll)  ← 跨平台串口库 (vendored)
                              ▼
              USB 串口 (4 Mbps)
                              ▼
              调试板 (USB ↔ CAN-FD 桥)
                              ▼
              FDCAN 总线 → 各电机
```

要点: Python 侧的笛卡尔/动力学能力 (Pinocchio) 是**纯软件层**, 不下沉到 C++ SDK;
原生 C++ SDK 只做关节空间 + 夹爪, 笛卡尔规划交给 Python (wrs / Pinocchio) 算完后把关节路径喂下来执行。

## 4. 协议栈与数据流

### 4.1 物理链路

```
PC ──USB 虚拟串口(4Mbps)──▶ 调试板 ──FDCAN──▶ 电机
```

调试板是一个 **USB↔CAN-FD 桥**。PC 发出的不是裸 CAN 帧, 而是符合调试板自定义封装的串口帧;
调试板再转成 FDCAN 帧下发到电机。因此 `set_*` 指令的字节布局必须严格匹配调试板结构 (历史上的"FDCAN 协议不匹配"问题即源于此)。

### 4.2 收发路径

- **发送**: 高层 API → `HightorqueSerial::set_*` 构造串口帧 → serial_cmake 写串口。
- **接收**: `enable_async_rx()` 启一个**异步 RX 线程**持续读串口、解析电机状态帧、刷新缓存;
  `start_state_polling(ids, hz)` 再启一个**后台轮询线程**按固定频率主动请求状态。
  高层 `get_motor_states(prefer_cache=True)` 默认读缓存, 不阻塞控制环。

### 4.3 单位换算 (贯穿全栈)

| 层 | 单位 |
|---|---|
| 用户 API (默认) | 弧度 rad (或 `is_radians=False` 时度) |
| 协议原生 | **圈 turns** (`1 turn = 2π rad`) |
| 力矩命令 | raw int16 |
| 软限位 (cfg) | `pos_unit` 指定 (`turns`/`radians`/`degrees`), 解析时归一为圈 |

Python 侧 `_rad_to_turns` / `_turns_to_rad`, C++ 侧 `to_turns` / `from_turns` 负责转换。

## 5. 核心模块详解

### 5.1 协议层 `HightorqueSerial` (C++, vendored)

驱动核心, 双侧共用。`bindings.cpp` 把它原样导出给 Python。主要能力:

| 类别 | 方法 (节选) | 说明 |
|---|---|---|
| 连接 | 构造 `(port, baud)` / `enable_async_rx` / `start_state_polling` | 串口 + 异步 RX + 轮询 |
| 模式 | `set_motor_mode(id, mode)` | 写电机寄存器 `0x00` |
| 位置控制 | `set_pos_vel_acc` | 位 + 速 + 加速度 (单关节 S 段) |
| 力限控制 | `set_pos_vel_tqe` | 位 + 速 + 最大力矩 (力控抓取) |
| 多电机同步 | `set_many_pos_vel_tqe` | 一拖多 (CAN ID `0x8090`), `move_j` 多关节 / `servo_j` |
| MIT | `set_pos_vel_tqe_kp_kd` | 位+速+力矩+Kp+Kd (留底层, 这批电机固件不响应) |
| 纯量 | `set_velocity` / `set_torque` / `set_current` / `set_voltage` | 单一物理量 |
| 软限位 | `enable_position_limit` / `apply_limits_to` | 下发软限位, 发送时自动 clamp |
| 看门狗 | `set_timeout(id, ms)` | 固件级超时自动 brake |
| 状态 | `read_motor_state` / `get_cached_state` | 同步读 / 读缓存 |

### 5.2 pybind11 绑定 `panthera_motor`

`bindings.cpp` 导出:

- **枚举**: `PosUnit`, `CanFault`
- **结构体**: `MotorState`, `CanStatus`, `PortInfo`, `Stats`, `RobotConfig`, `ManyMotorCmd`
- **函数**: `list_serial_ports`, `find_likely_debug_boards`, `to_turns`, `from_turns`, `parse_motor_state_int16`
- **主类**: `HightorqueSerial`

`.pyd` 按 Python ABI 编译, 文件名 `panthera_motor.cpXY-win_amd64.pyd` 必须匹配当前解释器 (cp310 / cp38)。改 `bindings.cpp` 或协议层后必须重新构建。

### 5.3 高层控制器 `FafuRobotController` (Python)

主入口, `fafu_robot_controller.py`。构造时:

```python
FafuRobotController(
    cfg_path,            # 必填: robot.cfg 路径
    port=None,           # 覆盖 cfg.port
    baudrate=None,       # 覆盖 cfg.baudrate
    has_gripper=False,   # 是否有夹爪
    gripper_motor_id=None,
    auto_enable=True,    # 构造即进 0x0A 主动模式
    auto_polling=True,   # 启后台状态轮询
    async_rx=None,       # None=按 cfg.use_async_rx
)
```

构造流程: 加载 cfg → 拆分关节/夹爪电机 ID → 打开串口 → `apply_limits_to` 下发软限位 → 通信预检 (`_precheck_communication`) → `enable()` → 启异步 RX → 启状态轮询。

支持上下文管理器 (`with ... as arm:`), 退出自动 `close_connection`。

### 5.4 原生 C++ SDK `fafu_robot::FafuRobotController`

从 Python 版 1:1 移植, 用 `Options` / `MoveOpts` / `GripperOpts` / `GraspOpts` struct 代替 Python 关键字参数。范围: 关节空间 + 夹爪 (含力控抓取 + servoJ), **不含**笛卡尔 / IK / Pinocchio。析构自动 `close_connection(joints=Stop, gripper=Brake)`。

## 6. 关键概念

### 6.1 电机模式 (mode 字节, 写寄存器 `0x00`)

| 常量 | 字节 | 含义 | 行为 |
|---|---|---|---|
| `MODE_STOP` | `0x00` | 停止 / 自由 | PWM 关断, 可手推, 不耗电 |
| `MODE_ACTIVE` / `MODE_POSITION` | `0x0A` | 主动控制 | 闭环, 接受位/速/力/MIT 各种命令 |
| `MODE_BRAKE` | `0x0F` | 短路刹车 | 三相短路, 抗扭, 不出力, 不耗电 |

> 位置/速度/力矩/MIT 都共用 `0x0A`, 区别在子帧字段而非 mode 字节。`MODE_POSITION` 是 `MODE_ACTIVE` 的旧别名。

### 6.2 力矩换算 (⚠️ 关键)

固件力矩命令是 raw int16, **LSB = `coeff * 0.01` Nm/count**:

```
raw = round(tau_nm / (coeff * 0.01))
```

`coeff` 取自每关节电机型号的 `TORQUE_COEFF` 表 (C++ `hightorque_serial.cpp` 与 Python `fafu_robot_controller.py` 两份必须一致):

| 型号 | coeff | 关节 |
|---|---|---|
| `M5036_02` | 0.67 | J1, J4 |
| `M6036_02` | 0.677 | J2, J3 |
| `M4438_30` | 0.5256 | J5, J6, J7 |

> 历史 bug: 早期漏了 `*0.01`, 导致 raw 偏小 100 倍, 需要用 `torque_scale≈90` 硬凑; 修复后正常 `torque_scale≈1.0`。**改 coeff 后必须重编 `.pyd`。**

### 6.3 速度 / 速度参数

`speed` 参数是 0–100 的百分比, 内部映射到平均速度 (turns/s, 上限 `_VEL_AVG_MAX_TPS = 0.5`)。`servo_j` 的 `max_vel` 直接是 rad/s。

## 7. 功能模块

### 7.1 关节空间运动

- `move_j(angles, is_radians=True, speed=50, block=True, tolerance=0.01)`: 离线 **S 曲线**插值 (`_move_scurve`), 多关节用 `set_many_pos_vel_tqe` 一拖多同步; 阻塞模式等到位 (容差判定 + `_SETTLE_MS` 沉降)。
- `go_home(speed, block)`: 回零位。
- `move_jntspace_path(path, speed)`: 走关节路点路径, 装了 `wrs` (TOPPRA) 时做时间最优插值。
- 状态读取: `get_joint_values()` (rad), `get_joint_velocities()` (rad/s), `get_motor_states()` (dict)。

### 7.2 servoJ 在线流式控制

给上层在线规划 / teleop / 视觉伺服用, 典型 100–200 Hz:

```
servo_start(ServoOpts) → 循环 servo_j(target) → servo_end(finish_mode)
```

四道安全防线:

| # | 防线 | 默认 |
|---|---|---|
| 1 | **固件看门狗** `watchdog_ms` (最关键) | 100 ms |
| 2 | 单步限幅 `max_step_rad` | 0.05 rad |
| 3 | 跟踪误差 `max_lag_rad` (累计 lag_count, 不立即停) | 0.2 rad |
| 4 | 软限位 (复用 cfg) | 取自 cfg |

`ServoOpts` 还支持 UR-servoj 风格的 `feedforward_vel` (速度前馈, 减少高频啸叫)、`lookahead_time` (一阶低通平滑)、`lag_abort_consecutive` (连续 lag 自动停)。看门狗保证上位机崩溃 / Ctrl-C / 拔 USB 时电机自动 brake。

### 7.3 夹爪与力控抓取

| 方法 | 语义 |
|---|---|
| `open_gripper(effort=None)` | 张到上软限位 |
| `close_gripper(effort=None)` | 关到下软限位 |
| `gripper_control(angle, effort)` | Piper 风格任意角度 + 力限 (`set_pos_vel_tqe`) |
| `grasp(force_threshold, effort, vel, timeout)` | **力控抓取** → `GraspResult` |

`grasp()` 在 Python 侧做力矩监测状态机, 早停判定: 力矩到阈值 (`detected_object_force`) / 闭合后失速 (`detected_object_stall`) / 到位无阻力 (`reached_target`) / 没动 (`no_movement`) / 超时 (`timeout`)。`GraspResult` 含 `grasped` / `reason` / `angle_rad` / `closed_deg` / `peak_torque_raw` / `duration_s`。标定脚本见 `tests/test_fafu_grasp_calibrate.py`。

### 7.4 软限位 / 急停

`set_limit` / `get_limit` / `disable_limit` / `clear_limits` / `emergency_stop` / `resume`。软限位下发到驱动层, 发送时自动 clamp。

### 7.5 动力学 (Pinocchio, 可选)

`setup_dynamics(urdf_path=None, motor_models, tau_limit, torque_scale, friction, eef_frame)` 加载 URDF 刚体模型 (默认找包内 `fafu_robot_description/*.urdf`, 末端帧默认 `tool_link`)。需要 `pinocchio` (Windows 上建议 conda-forge / WSL; 缺失时动力学方法报清晰错误, 其余功能不受影响)。

**正/逆运动学与笛卡尔运动:**

- `forward_kinematics(q)` / `get_pose()`: FK, 返回位姿。
- `inverse_kinematics(...)`: 阻尼最小二乘 (DLS) + 多初值策略。
- `move_p(...)` / `move_l(...)`: 笛卡尔点到点 / 直线 (SE3 测地线插值), 支持 rad/deg。

**力矩项:**

- `get_gravity(q)` 重力 G(q), `get_mass_matrix` 质量阵, `get_coriolis`, `get_dynamics`。
- `get_friction_compensation`: 库仑 + 粘滞摩擦 (`FrictionParams`, 低速死区抑制抖振)。
- `compute_compensation_torque` / `apply_compensation_torque` / `tau_to_raw` (干跑预览 raw)。

**重力补偿 / 拖动示教** (`start_gravity_compensation`):

纯重力 + (可选) 摩擦前馈 + 阻抗 (K/B/I) 软保持, 实现"拖到哪停到哪"。默认每关节增益:

```
K = (1.5, 8.0, 10.0, 10.0, 1.5, 1.5)   # 软保持刚度
B = (0.4, 0.4, 0.6, 0.8, 0.2, 0.2)     # 阻尼
I = (0.0, 2.5, 3.5, 6.0, 0.0, 0.0)     # 积分 (只给承重关节, 抵消静摩擦下垂)
```

含积分抗饱和、速度低通、失控保护 (runaway guard)、进入拖动消抖。Ctrl+C 退出序列: 刹车 → (可选间隔) → 慢速回零 → 刹车。

两个对照测试脚本:
- `test_fafu_gravity_only.py`: 严格照搬厂商纯重力脚本 (无摩擦无阻抗, ~500Hz)。J4 重力力矩天生最小, 默认 `torque_scale=1.0,1.0,1.0,1.3,1.0,1.0` 单独补 J4 抗下垂。
- `test_fafu_gravity_comp.py`: 完整重力 + 摩擦 + 阻抗拖动示教。

## 8. 配置 `robot.cfg`

INI / `key=value` 格式 (`#` 或 `;` 注释)。关键字段:

| 字段 | 默认 | 含义 |
|---|---|---|
| `port` | `auto` | `auto` 自动扫 USB / 或 `COM14` / `/dev/ttyUSB0` |
| `baudrate` | `4000000` | 串口波特率 (4 Mbps) |
| `motor_ids` | `1..7` | 关节列表 (夹爪 ID 在构造时单独指定) |
| `pos_unit` | `degrees` | 限位单位, 解析时归一为圈 |
| `limits.<id>` | 见文件 | 每关节软限位 `lo, hi` (按 pos_unit) |
| `control_rate_hz` | `250` | 高频控制环目标频率 (250Hz 是甜蜜点) |
| `max_torque_raw` | `0` | 一拖多 tqe (0=用电机默认最大力矩) |
| `use_async_rx` | `true` | 异步 RX 线程 (100Hz 模式) |
| `trajectory_dt_s` | `4.0` | 单次目标运动时长 |

> ⚠️ `limits.*` 是保守初值, **必须按实际机械行程修改**。

## 9. 构建与部署

### 9.1 Python 侧 (大多数用户)

仓库已带 cp310 / cp38 预编译 `.pyd`。匹配的话:

```bash
cd fafu_robot_sdk/fafu_robot_python
pip install -r requirements.txt
python tests/smoke_test.py
```

### 9.2 C++ 侧 (换 Python 版本 / 改 binding / 写 C++ 程序)

```bat
REM 前置: VS2019/2022 + CMake 3.18+ + pybind11
cd fafu_robot_sdk\fafu_robot_cpp
build.bat                 REM Release, 编 .pyd + .lib + 例程, 自动 copy 到 python 侧
build.bat Debug
```

可选 CMake 选项: `FAFU_BUILD_PYBIND11_MODULE` / `FAFU_BUILD_NATIVE_SDK` / `FAFU_BUILD_SDK_EXAMPLES` (默认全 ON)。

> `.pyd` 按 ABI 编译: 切到目标 conda 环境再 `build.bat`, 输出名即该环境 ABI tag。

### 9.3 自包含与可移植

整个 `fafu_robot_sdk/` 拷到任何地方都能独立构建。协议层 + serial_cmake 已 vendor; 上游升级时按 `fafu_robot_cpp/README.md` 的对照表覆盖回来重编即可。

## 11. 已知限制 / 排错

### 限制

- MIT 通道 (`set_pos_vel_tqe_kp_kd`) 这批电机固件不响应, 力矩走 `set_torque` (0x0A)。
- 原生 C++ SDK 不含笛卡尔 / IK (留给 Python)。
- 同一时刻只能一个进程占串口 (Python 在跑则 C++ exe 启动会 port busy)。
- `pinocchio` 在 Windows pip 安装不稳, 建议 conda-forge / WSL。

### 常见问题

| 现象 | 处理 |
|---|---|
| `ImportError: DLL load failed` | `serial_cmake.dll` 不在本目录 / 缺 VC++ Redistributable |
| `ModuleNotFoundError: panthera_motor` | `.pyd` ABI 不匹配, 去 `fafu_robot_cpp/` 重 build |
| 串口打开失败 | cfg `port=auto` 或写死 `COMxx`; 端口被占 |
| 所有电机 `mode 0 / fault≠0` | CAN 未通: 查 24V / 终端电阻 / `motor_ids` |
| `motors [...] did not respond within 500ms` | CAN 通信瞬断 / 电机未上电 / ID 配错 |
| 夹爪不响应 | 确认 `has_gripper=True, gripper_motor_id=N` 且 N 在 `motor_ids` |
| 重力补偿"没力气" / 需要 `torque_scale≈90` | 旧版 `*0.01` 力矩 bug, 用修复后的 `.pyd`, `torque_scale≈1.0` |
| J4 纯重力下垂 | J4 重力力矩天生最小, 单独补 `torque_scale` 1.3 或用 `gravity_comp` (带摩擦补偿) |

## 12. 与 piper.py 的对照

| Piper | Fafu | 备注 |
|---|---|---|
| `PiperArmController(can_name="can0")` | `FafuRobotController(cfg_path="robot.cfg")` | CAN vs USB→CAN-FD 调试板 |
| `move_j(angles, speed=)` | 同名同语义 | ✅ |
| `move_p / move_l` | ✅ (Pinocchio FK/IK) | Fafu 早期 NotImplemented, 现已实现 |
| `open_gripper(width)` | `open_gripper(angle)` | Fafu 夹爪是旋转关节, 单位是角度 |
| `close_gripper(effort=N·m)` | `close_gripper(effort=raw int16)` | effort 单位不同 |
| ─ | `grasp(force_threshold=...)` | Fafu 独有: Python 侧力矩监测早停 |
