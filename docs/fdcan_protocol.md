## 1. CAN-FD 与 ID

### 1.1 波特率

| 段 | 说明 |
|----|------|
| 仲裁段 | 1 Mbps |
| 数据段 | 最高 5 Mbps，也可用 1 Mbps（可变波特率） |

### 1.2 16 位 ID

- **0x7F** 为广播地址（以固件与例程为准）。
- **高 8 位（源地址）**
  - **最高位 = 1**：需要回复（总开关）；若开启但不发查询指令，可能返回数据段长度为 0 的帧。
  - **最高位 = 0**：无需回复。
  - 其余 7 位：信号源地址。
- **低 8 位（目的地址）**
  - 最高位为 0。
  - 其余 7 位：目的地址。

**示例**

| ID | 含义 |
|----|------|
| `0x8001` | 源地址 0 → 目的地址 1，且**需要回复** |
| `0x0100` | 源地址 1 → 目的地址 0，**无需回复** |

### 1.3 扩展帧与标准帧

| 方向 | 说明 |
|------|------|
| 主机 → 电机（指令） | 扩展帧、标准帧均可用于控制指令 |
| 主机 → 电机（读状态） | **需使用扩展帧** |
| 电机 → 主机（反馈） | ID 1–7：**标准帧**；ID ≥ 8：**扩展帧** |

## 2. 协议基本约定

1. **最小单位是子帧**：每条子帧可向一个或多个寄存器读/写。
2. **一帧可含多个子帧**：一条 CAN-FD 数据区可由多个子帧拼接。
3. **功能通过写寄存器实现**：在一条 FDCAN 帧中写入一个或多个寄存器值。
4. **数据类型**：`int8` / `int16` / `int32` / `float`。
5. **同一条子帧内**数据类型必须相同；**不同子帧**可不同。
6. **小端序**：低字节先发。
7. **填充**：帧尾需凑长度时用 **`0x50`** 表示 NOP（无操作）。
8. **“无限制”取值**

| 类型 | 值 |
|------|-----|
| int8 | `0x80` |
| int16 | `0x8000` |
| int32 | `0x80000000` |
| float | `NAN` |

9. **模式三（一拖多）**：由 FDCAN ID **低 8 位**识别。  
   - 低 8 位 **≥ 0x80**：一拖多模式。  
   - 低 8 位 **< 0x80**：模式一或模式二（普通模式）。

## 3. 子帧格式

### 3.1 模式一（发送）

```c
uint8_t tdata[] = { cmd, addr, a1, a2, b1, b2, ... };
```

**cmd 字节**

| 位域 | 含义 |
|------|------|
| `[7:4]` | 读/写/回复：`0000` 写，`0001` 读，`0010` 回复 |
| `[3:2]` | 数据类型：`00` int8，`01` int16，`10` int32，`11` float |
| `[1:0]` | 个数：`01` 1 个，`10` 2 个，`11` 3 个，`00` 表示**模式二** |

- **addr**：起始寄存器地址。
- **数据**：按 addr、addr+1、… 依次写入。

### 3.2 模式二（发送）

```c
uint8_t tdata[] = { cmd, num, addr, a1, a2, b1, b2, ... };
```

- cmd 低两位固定为 `00`，表示模式二；**num** 单独表示数据个数。
- 实质：模式一在“个数”写为 0 时，用下一字节表示个数，其余后移。

### 3.3 模式三（一拖多）

- 由 ID 低 8 位决定；**一拖多时数据类型均为 int16**。
- 帧内布局：**电机 1 数据、电机 2 数据、…**，末尾 **2 字节**为查询电机状态指令。

**例：ID `0x8090`（一拖多 位置+速度+力矩）**

```c
uint8_t cmd[] = {
  pos11, pos12, vel11, vel12, tqe11, tqe12,
  pos21, pos22, vel21, vel22, tqe21, tqe22,
  /* 占位 */, 0x17, 0x01
};
```

**例：ID `0x8093`（一拖多 位置+速度+力矩+PD）**

每套电机占 10 字节（位置、速度、力矩、rKp、rKd 各 int16），多电机依次排列，最后 `0x17, 0x01` 等查询尾。

### 3.4 接收协议

接收模式与发送时选用的模式一致（模式一 / 模式二结构对应）。

## 4. 收发示例

### 4.1 发送（模式一）

```c
uint8_t cmd[] = {
  0x01, 0x00, 0x0A,
  0x0A, 0x20, 0x00,0x00,0x00,0x80, 0x10,0x27,0x00,0x00,
  0x50, 0x50, 0x50
};
```

- 子帧 1：写寄存器 `0x00` = `0x0A`（进入位置模式）。
- 子帧 2：写 `0x20` 位置 = `0x80000000`（无限制），写 `0x21` 速度 = `0x2710`（按 int32 LSB 为 0.1 转/秒，以寄存器表为准）。
- `0x50`：填充至 CAN-FD DLC。

### 4.2 发送（模式二）

与上类似，第二子帧以 `0x08` 开头且带 `num=2`。

### 4.3 接收（模式二示例）

```c
uint8_t rdata[] = {
  0x28, 0x04, 0x00,
  0x0A,0x00,0x00,0x00,
  0x96,0x1D,0x04,0x00,  /* 位置示例值 */
  0x54,0x40,0x02,0x00,  /* 速度示例值 */
  0x86,0x01,0x00,0x00,  /* 力矩示例值 */
  0x50
};
```

具体数值换算见下节「单位」与寄存器说明。

## 6. 扭矩系数

- **d**：摩擦等偏置；精度要求不高可忽略；**fdcan_h730 工程自 v3.0.5 起文档说明不再加入 d 项**。

## 7. 单电机控制模式与 API

封装函数以 `FDCAN_HandleTypeDef` + `id` 为主（与 `livelybot_fdcan` 一致）。每种模式通常提供 **float / int16 / int32** 三种重载。

| 模式 | 说明 | 典型 API |
|------|------|----------|
| DQ 电压 | Q 相电压控制 | `set_dq_volt_*` |
| DQ 电流 | Q 相电流控制 | `set_dq_current_*` |
| 位置 | 最大速度/加速度冲到目标位，**较猛** | `set_pos_*` |
| 速度 | 最大加速度到目标转速 | `set_vel_*` |
| 力矩 | 目标力矩 | `set_torque_*` |
| 位置+速度+力矩 | 以设定速度走向目标位 | `set_pos_vel_tqe_*` |
| 位置+速度+**最大**力矩 | 同上且限力矩；力矩过小可能达不到目标速度 | 同名 `set_pos_vel_tqe_*`（与文档分类一致） |
| **梯形控制** | 加速→匀速→减速，**推荐一般位置控制** | `set_pos_velmax_acc_*` |
| 速度+加速度 | 带加速度限制的速度控制 | `set_vel_acc_*` |
| **运控（MIT）** | `τ = kp·Δp + kd·Δv + τ_ff`，建议用 **`_2`** | `set_pos_vel_tqe_kp_kd_*_2` |
| 读状态 | 模式、错误、位置、速度、力矩等 | `read_motor_state_*` |
| 读版本 | 固件版本 | `read_motor_version_int16` |
| 停止 | 三相悬空，可自由转动 | `set_motor_stop_*` |
| 刹车 | 短接相线，阻尼制动 | `set_motor_brake_*` |
| 软重启 | 无反馈，重启后停止模式 | `set_motor_reset_int8` |

### 7.1 位置模式注意

- 速度与力矩均为**最大**，电流可能瞬时 **5 A～10 A**；电源响应不足易报错。
- 一般位置建议用 **梯形控制** `set_pos_velmax_acc_*`。

### 7.2 运控（MIT）要点

```
输出力矩 = 位置偏差 × kp + 速度偏差 × kd + 前馈力矩
```

| 条件 | 行为提示 |
|------|----------|
| kp=0, kd=0 | 仅靠前馈力矩，可持续输出恒定力矩 |
| kp=0, kd≠0 | 可趋向匀速；有静差；kd 过大易振荡 |
| kp≠0, kd=0 | **易振荡，不宜** |
| kp≠0, kd≠0, vel=0 | 定点 holding |
| pos(t) 连续可导，vel=dp/dt | 轨迹跟踪 |

**旧版** `motor_set_pos_vel_tqe_kp_kd` / `set_pos_vel_tqe_kp_kd`（无 `_2`）：文档说明位置依赖积分，**低频易异常**。  
**建议**使用 **`_2`** 系列。

## 8. 一拖多模式（motor_many）

### 8.1 通用规则

- 仅 **`motor_many_send`** 真正发帧；其余函数只写缓存。
- 同一条 CAN 通道上，一拖多帧内各电机 **模式一致**。
- 仲裁 1 Mbps、数据 5 Mbps 时：约 **1 ms 内最多 10 个电机** → 1 kHz 下最多 10 轴；更多轴需降频。
- **TINT16** 量程约 **±3.2 圈**（原文说明）。

### 8.2 使用流程

```c
motor_many_vel(PORT1, 1, 0.1f);
motor_many_vel(PORT1, 2, 0.1f);
motor_many_vel(PORT1, 3, 0.1f);
motor_many_send(PORT1);
```

### 8.3 API 一览（均需再 `motor_many_send`）

| 功能 | 函数 |
|------|------|
| DQ 电压 | `motor_many_dq_volt` |
| DQ 电流 | `motor_many_dq_current` |
| 位置 | `motor_many_pos` |
| 速度 | `motor_many_vel` |
| 力矩 | `motor_many_tqe` |
| 位置+速度 | `motor_many_pos_vel` |
| 位置+速度+最大力矩 | `motor_many_pos_vel_MAXtqe` |
| 梯形 | `motor_many_pos_vel_acc` |
| 运控 MIT | **`motor_many_pos_vel_tqe_kp_kd_2`**（推荐） |

参数 **portx**：`PORT1` / `PORT2` / `PORT3` 对应 CAN 通道 1/2/3。  
**type**：`TFLOAT` / `TINT32` / `TINT16` 影响精度与量程。

运控旧版 `motor_many_pos_vel_tqe_kp_kd` 不推荐；**请用 `_2`**。
